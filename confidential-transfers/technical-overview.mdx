## Core Architecture

Fairblock's confidential transfer system operates across three layers:

| Layer | Role | Technology |
| --- | --- | --- |
| **FairyRing** | Confidentiality execution layer | CosmWasm contracts, homomorphic calculations, ZK verification |
| **EVM Escrow Contracts** | Bridge endpoints | Minimal contracts that lock tokens and interface with cross-chain communication |
| **IBC Relaying** | Cross-chain transport | Trust-minimized light-client verification |

## Cryptographic Primitives

Fairblock uses **Twisted ElGamal** — a fast additive homomorphic encryption scheme — combined with lightweight ZK proofs.

Users prove two things without revealing the underlying values:

1. **"I'm allowed to transfer X"** — authorization proof
2. **"The math checks out"** — range proof and balance proof

<Note>
  Fairblock avoids fully homomorphic encryption (FHE) in favor of lightweight schemes that enable real-time transactions on mobile devices.
</Note>

## User Flows

### Deposit

```
User → Lock tokens on EVM chain → IBC relayer → Encrypted balance on FairyRing
```

Tokens are locked in escrow contracts on the source chain. An encrypted balance is created on FairyRing, visible only to the account owner.

### Confidential Transfer

```
Sender → Generate ZK proof + HE ciphertext → FairyRing verifies → Update encrypted balances
```

FairyRing performs homomorphic operations on encrypted balances. The ZK proof ensures the transfer is valid without revealing amounts.

### Withdrawal

```
User → Prove balance ownership → FairyRing → IBC relayer → Unlock tokens on destination chain
```

Users redeem encrypted balances back to native tokens on any supported chain.

## Anonymous Confidential Transfers

For enhanced privacy, Fairblock offers **anonymous transfers** that separate confidential accounts from public wallet addresses.

| Component | Role |
| --- | --- |
| **Fairycloak** | Relay layer for anonymous operations — submits transactions without linking to user wallets |
| **Fairyport** | Handles cross-chain message delivery |
| **Numeric Account IDs** | Replace EVM addresses to break the link between identity and activity |

  Anonymous transfers are currently in early access. [Apply for access](https://build.fairblock.network) through the Confidential Builders Program.
</Warning>
