title: "Fairates"

Fairates is a fixed-rate lending market that clears via a leaderless, sealed-bid, single-price auction.

## How It Works

| Property | Description |
| --- | --- |
| **No trusted auctioneer** | No single party can decrypt, censor, or peek at bids |
| **Decentralized key management** | Decryption keys are split among a decentralized validator set via MPC and DKG |
| **Sealed bids** | Bids stay confidential until the auction closes |
| **Uniform pricing** | Every winning lender and borrower gets the same fixed rate |

## Auction Flow

<Steps>
  <Step title="Bid Submission">
    Lenders and borrowers submit encrypted bids during the auction window. Bids are encrypted using the Master Public Key (MPK) generated by FairyRing validators.
  </Step>
  <Step title="Auction Close">
    When the auction window ends, FairyRing validators collaborate to derive the decryption key. No single validator can decrypt bids alone.
  </Step>
  <Step title="Price Discovery">
    All bids are decrypted simultaneously. The clearing rate is computed as a single uniform price — all winning participants get the same rate.
  </Step>
  <Step title="Settlement">
    Matched lenders and borrowers are settled onchain at the clearing rate.
  </Step>
</Steps>

## Why Sealed-Bid Auctions

Traditional DeFi lending markets suffer from:

- **Rate manipulation** — visible bids enable strategic undercutting
- **Information leakage** — competitors see your lending appetite and rate expectations
- **Front-running** — MEV bots can sandwich rate-setting transactions

Fairates eliminates these issues by keeping all bids confidential until simultaneous decryption.

## Cryptographic Foundation

Fairates uses **threshold Identity-Based Encryption (IBE)** with FairyRing's MPC-DKG infrastructure:

- **Encryption:** Bids are encrypted with IBE using the auction ID as the identity
- **Decryption:** Validators generate keyshares; the decryption key is derived when a threshold of validators contribute
- **Verification:** All operations are verifiable onchain
